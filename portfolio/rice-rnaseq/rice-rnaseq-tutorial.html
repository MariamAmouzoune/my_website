01

Background & Objective

Rice blast, caused by the ascomycete fungus Magnaporthe oryzae, is the most devastating disease of rice worldwide, threatening food security across Asia, Africa, and South America. Understanding the transcriptomic reprogramming that occurs during infection is critical for designing durable resistance strategies.

This project is a full reproduction of the RNA-seq analysis published by Cui et al. (2021) in ScienceDirect — "Changes in the epigenome and transcriptome of rice in response to Magnaporthe oryzae infection". Using the publicly deposited raw sequencing data (NCBI SRA accession SRP230704), I reproduced the complete bioinformatics pipeline from raw FASTQ files to a final list of differentially expressed genes (DEGs), then generated publication-quality visualizations.

Dataset: 8 paired-end RNA-seq libraries — 6 infected (M. oryzae-treated, three time points × two replicates) and 2 mock-inoculated controls. SRR IDs: SRR10498876–SRR10498883. Reference genome: Oryza sativa IRGSP-1.0 (Ensembl Plants release 59).
02

Analysis Pipeline

The pipeline was executed entirely on WSL2 (Ubuntu 22.04) for all pre-processing, alignment, and quantification steps, then switched to R for statistical modelling and visualisation. Each step below shows the exact commands used.

01
Data download
16 paired-end FASTQ files retrieved from EBI FTP using wget
02
Quality control
FastQC — per-base quality, adapter content, overrepresented sequences
03
Adapter trimming
fastp — paired-end mode, quality threshold Q ≥ 30
04
Splice-aware alignment
HISAT2 — aligned to IRGSP-1.0 index; samtools sort + index
05
Transcript quantification
Cufflinks per sample → cuffmerge → cuffdiff
06
Differential expression (R)
DESeq2 — filter (rowSums ≥ 5), size-factor normalisation, Wald test; LFC shrinkage via apeglm
07
Visualisation (R)
EnhancedVolcano · pheatmap · ggplot2 PCA · MA plot · Dispersion estimates
Step 01

Step 02

Quality Control — FastQC

FastQC generates per-sample HTML reports covering all key quality metrics. Three parameters were checked to decide on data quality before proceeding to trimming: (1) Per-base sequence quality, (2) Overrepresented sequences, and (3) Adapter content.

Bash — WSL2 · Step 02: Quality Control 
##──────────────────────────────────────────────────────────
## STEP 02: Quality Control using FastQC
## Install: sudo apt-get install fastqc
##──────────────────────────────────────────────────────────

# Run FastQC on all FASTQ files at once
fastqc *.fastq

# Three parameters to check before deciding on trimming:
# 1. Per base sequence quality  → should be green (Phred ≥ 28)
# 2. Overrepresented sequences  → flag if any sequence > 0.1%
# 3. Adapter content            → trim if adapters detected
Step 03

Adapter Trimming — fastp

If FastQC flags adapter contamination or low-quality bases, reads are trimmed with fastp in paired-end mode. After trimming, FastQC is run again on the trimmed files to confirm quality improvement before alignment.

Bash — WSL2 · Step 03: Trimming 
##──────────────────────────────────────────────────────────
## STEP 03: Trimming using fastp
## Install: sudo apt-get install fastp
##──────────────────────────────────────────────────────────

# Paired-end trimming — one sample example
fastp -i SRR10498876_1.fastq -o trim_SRR10498876_1.fastq       -I SRR10498876_2.fastq -O trim_SRR10498876_2.fastq       --adapter_fasta adapter.fasta

# Loop over all samples
for sample in control1 control2 Treated1R1 Treated1R2                      Treated2R1 Treated2R2 Treated3R1 Treated3R2; do
    fastp -i  ${sample}_1.fastq -o  trim_${sample}_1.fastq           -I  ${sample}_2.fastq -O  trim_${sample}_2.fastq           --adapter_fasta adapter.fasta
    echo "✅  Trimmed: ${sample}"
done

# Re-check quality after trimming
fastqc trim_*.fastq
Step 04

Splice-Aware Alignment — HISAT2

Trimmed reads are aligned to the Oryza sativa IRGSP-1.0 reference genome using HISAT2 with the --dta-cufflinks flag (required for downstream Cufflinks quantification). The output is piped directly into samtools sort to produce sorted BAM files in one pass, then indexed for fast random access.

Bash — WSL2 · Step 04A: Download reference genome & build index 
##──────────────────────────────────────────────────────────
## STEP 04A: Reference genome and HISAT2 index
## Install: sudo apt-get install hisat2 samtools
##──────────────────────────────────────────────────────────

# Download Oryza sativa IRGSP-1.0 reference genome (Ensembl Plants release 59)
wget ftp://ftp.ensemblgenomes.org/pub/plants/release-59/fasta/oryza_sativa/dna/Oryza_sativa.IRGSP-1.0.dna.toplevel.fa.gz
wget ftp://ftp.ensemblgenomes.org/pub/plants/release-59/gtf/oryza_sativa/Oryza_sativa.IRGSP-1.0.59.gtf.gz

# Decompress
gunzip Oryza_sativa.IRGSP-1.0.dna.toplevel.fa.gz
gunzip Oryza_sativa.IRGSP-1.0.59.gtf.gz

# Build HISAT2 genome index (required once; speeds up all alignments)
hisat2-build Oryza_sativa.IRGSP-1.0.dna.toplevel.fa rice_index
Bash — WSL2 · Step 04B: Align all samples (loop) 
##──────────────────────────────────────────────────────────
## STEP 04B: Map all 8 samples — sorted BAM files in one pass
## --dta-cufflinks: required for Cufflinks downstream
## -p 6           : use 6 CPU threads
##──────────────────────────────────────────────────────────

for sample in control1 control2 Treated1R1 Treated1R2                      Treated2R1 Treated2R2 Treated3R1 Treated3R2; do

    hisat2 --dta-cufflinks -p 6 -x rice_index            -1 ${sample}_1.fastq -2 ${sample}_2.fastq     | samtools sort -o sorted_${sample}.bam

    samtools index sorted_${sample}.bam
    echo "✅  Mapped and sorted: ${sample}"

done

# Optional: convert BAM to SAM to inspect alignment headers
samtools view sorted_control1.bam > sorted_control1.sam
head -50 sorted_control1.sam
Step 05

Transcript Quantification — Cufflinks Suite

Each sorted BAM file is processed individually with cufflinks to estimate transcript abundances. The per-sample GTF files are then merged with cuffmerge into a single consensus annotation, and cuffdiff performs the pairwise differential expression test between treated and control groups.

Bash — WSL2 · Step 05A: Cufflinks — per-sample quantification 
##──────────────────────────────────────────────────────────
## STEP 05A: Run Cufflinks on each sorted BAM
## -G : use reference annotation (guided assembly)
## -p : threads
##──────────────────────────────────────────────────────────

for sample in control1 control2 Treated1R1 Treated1R2                      Treated2R1 Treated2R2 Treated3R1 Treated3R2; do
    cufflinks -o cufflinks_${sample} -p 2               -G Oryza_sativa.IRGSP-1.0.59.gtf               sorted_${sample}.bam
    echo "✅  Cufflinks done: ${sample}"
done
Bash — WSL2 · Step 05B: Cuffmerge — merge transcripts 
##──────────────────────────────────────────────────────────
## STEP 05B: Create assembly.txt and run cuffmerge
##──────────────────────────────────────────────────────────

# Create the list of per-sample transcript GTF paths
for sample in control1 control2 Treated1R1 Treated1R2                      Treated2R1 Treated2R2 Treated3R1 Treated3R2; do
    echo "cufflinks_${sample}/transcripts.gtf" >> assembly.txt
done

# Verify the file
cat assembly.txt

# Merge all transcripts into a single consensus annotation
cuffmerge -o cuffmerge_output -p 2           -g Oryza_sativa.IRGSP-1.0.59.gtf           -s Oryza_sativa.IRGSP-1.0.dna.toplevel.fa           assembly.txt
Bash — WSL2 · Step 05C: Cuffdiff — differential expression 
##──────────────────────────────────────────────────────────
## STEP 05C: Cuffdiff — Treated vs Control
## -L : group labels
## -u : multi-read correction
##──────────────────────────────────────────────────────────

cuffdiff -o cuffdiff_output -p 2          -L Control,Treatment          -u cuffmerge_output/merged.gtf          sorted_control1.bam,sorted_control2.bam          sorted_Treated1R1.bam,sorted_Treated1R2.bam,sorted_Treated2R1.bam,sorted_Treated2R2.bam,sorted_Treated3R1.bam,sorted_Treated3R2.bam
Step 06

Differential Expression — DESeq2 (R)

The count table (count_table.csv) is loaded into R and analysed with DESeq2. Genes with a total count below 5 are filtered out before statistical testing. Significant DEGs are defined as adjusted p-value < 0.05 and |log₂FC| ≥ 1. LFC shrinkage via apeglm reduces noise from lowly expressed genes before visualisation.

R · Step 06A: Load data, build DESeq2 object, run pipeline 
##──────────────────────────────────────────────────────────
## STEP 06A: DESeq2 pipeline
##──────────────────────────────────────────────────────────
library(DESeq2)
library(ggplot2)

setwd("C:/Users/DELL/Desktop/RNA-seq_Analysis")

# Read count table (semicolon-separated)
Counts <- read.delim("count_table.csv", row.names = 1,
                      header = TRUE, sep = ";")

# Filter: keep genes with total count >= 5 across all samples
dat <- Counts[which(rowSums(Counts) >= 5), ]
dat <- as.matrix(dat)

# Define condition factor: 6 Treated, 2 Control; Control = reference
condition <- factor(c(rep("T", 6), rep("C", 2)))
condition  <- relevel(condition, ref = "C")

# Build coldata metadata frame
coldata <- data.frame(row.names = colnames(dat), condition)

# Create DESeqDataSet and run full pipeline
dds <- DESeqDataSetFromMatrix(countData = dat,
                               colData   = coldata,
                               design    = ~ condition)
dds <- DESeq(dds)
R · Step 06B: Extract results and significant DEGs 
##──────────────────────────────────────────────────────────
## STEP 06B: Extract results — Treated vs Control
##──────────────────────────────────────────────────────────

# Extract results (T vs C; C is reference)
deseq_results <- results(dds, contrast = c("condition", "T", "C"))
deseq_results <- as.data.frame(deseq_results)
deseq_results$GeneName <- row.names(deseq_results)

# Reorder columns for clarity
deseq_results <- subset(deseq_results,
  select = c("GeneName", "padj", "pvalue", "lfcSE",
             "stat", "log2FoldChange", "baseMean"))

# Write full results table
write.table(deseq_results, file = "deseq_results.tsv",
            row.names = FALSE, sep = "	")

# Extract significant DEGs: padj < 0.05 and |log2FC| >= 1
DEG <- subset(deseq_results, padj < 0.05 & abs(log2FoldChange) >= 1)
DEG <- DEG[order(DEG$padj), ]
dim(DEG)   # number of significant DEGs

write.table(DEG, file = "deseq_DEG.tsv",
            row.names = FALSE, sep = "	")

Quality Control: Dispersion Estimates

Before interpreting differential expression results, DESeq2's dispersion modelling must be evaluated. The plot below confirms a well-fitted negative binomial model: gene-wise estimates (black) shrink toward the fitted curve (red), and final shrunken estimates (blue) follow the expected mean-dispersion trend.

Figure 1 — Dispersion estimates
DESeq2 dispersion plot. Black dots: per-gene maximum-likelihood estimates. Red curve: fitted dispersion trend. Blue circles: final MAP-shrunken estimates used for testing. The expected decreasing trend with mean expression confirms robust model fitting with these 8 samples.
🔬
Interpretation
The dispersion estimates behave as expected for bulk RNA-seq data: high dispersion at low counts, converging to lower values at high expression. No unusual outlier clusters are visible, indicating good library quality and consistent sample preparation.

Sample Clustering: PCA

Principal component analysis (PCA) on VST-transformed counts reveals the global structure of the experiment and confirms that treatment explains the dominant source of variation.

Figure 2 — PCA of VST-transformed counts
PC1 explains 90% of total variance and cleanly separates infected (Treated) from control (C1, C2) samples along the x-axis. PC2 (5%) captures within-treatment variation, likely reflecting the three infection time points. The two control replicates cluster tightly, confirming low technical noise.
📊
Interpretation
The strong separation on PC1 validates the experimental design: the transcriptomic response to M. oryzae infection dominates the signal. Time-point variation within treated samples (visible along PC2) suggests a dynamic, time-dependent transcriptional response, consistent with staged infection biology.

Differential Expression: Volcano & MA Plots

Two complementary views of the differential expression results. The volcano plot combines statistical significance with fold change; the MA plot shows LFC versus mean expression to assess whether the LFC shrinkage was effective.

Figure 3 — Volcano plot (EnhancedVolcano)
Infected vs Control. Red: upregulated (log₂FC ≥ 1, p < 0.05). Blue: downregulated (log₂FC ≤ −1, p < 0.05). Grey: not significant. Total = 26,167 variables. Notable top genes labelled with connectors: Os01g0389200, Os01g0940700, and Os01g0198300 show the strongest upregulation.
Figure 4a — MA plot (shrunken LFC)
Blue dots: significant DEGs (padj < 0.05). Shrinkage via apeglm pulls extreme LFC estimates at low counts toward zero, reducing noise from lowly expressed genes.
Figure 4b — Top gene: Os07g0518100
Normalized counts for the gene with the highest log₂FC. Expression is near-zero in controls (C) but strongly induced across treated replicates (T), validating the DESeq2 call.

Heatmaps: Top DEGs & Z-score Profiles

Two heatmaps explore DEG expression patterns across all 8 samples: one using log₂-transformed normalised counts (top 10 DEGs) and one using a broader Z-score scaled view of defence-related genes.

Figure 5a — Top 10 DEGs (log₂ counts)
Hierarchical clustering of the 10 most significant DEGs. Treated samples (left cluster) show consistently higher expression for most top genes, particularly Os06g0633100 and Os01g0940700.
Figure 5b — Normalised counts overview
Broader view across a larger set of DEGs. The clear separation between treated and control columns confirms robust differential expression without requiring Z-score transformation.
Figure 6 — Z-score heatmap (defence & stress genes)
Z-score normalised expression across selected defence- and stress-related genes, annotated with gene names (e.g. OsCCT36, OsPR1b, OsDREB1C, PR1A). Red = high relative expression; blue = low. Treated replicates (Treated2R1–Treated1R2) form a clear high-expression cluster, while both controls and Treated1R1 show predominantly suppressed profiles, suggesting heterogeneity in early infection response.
🌾
Biological interpretation
The heatmap reveals induction of canonical defence genes including PR1A, OsPR1b, and OsPR10a (pathogenesis-related proteins), as well as transcription factors (OsDREB1C) and secondary metabolite biosynthesis genes. This pattern is consistent with the salicylic acid and jasmonate signalling cascades activated during incompatible rice–blast interactions described in the original publication.

Conclusions & Skills Demonstrated

This reproduction study successfully recapitulated the main findings of Cui et al. (2021): a large-scale transcriptomic reprogramming in rice upon M. oryzae infection, characterised by strong induction of pathogenesis-related genes and suppression of growth-related processes. The PCA, volcano plot, and heatmap patterns are qualitatively consistent with the published results.
